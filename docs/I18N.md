# Internationalization (i18n)

Multilingual from day one: **Arabic (default)** + **English**, with automatic
RTL/LTR. The architecture supports adding any number of languages through
configuration — no architectural changes.

## Moving parts

| Concern | File |
| --- | --- |
| Locale list, default, URL strategy | `src/i18n/routing.ts` |
| Per-request messages/locale | `src/i18n/request.ts` |
| Locale-aware `Link`/`useRouter` | `src/i18n/navigation.ts` |
| Locale detection + prefixing | `src/middleware.ts` |
| Locale metadata (dir, font, label) | `src/config/locales.ts` |
| UI strings (ICU) | `messages/<code>.json` |

## URLs & SEO

- Locale-prefixed routes (`/ar`, `/en`) via `localePrefix: "always"` → clean,
  crawlable localized URLs.
- `<html lang dir>` is set from the active locale in `app/[locale]/layout.tsx`.
- Localized metadata + Open Graph locale in `generateMetadata`.
- **TODO** (roadmap): emit `hreflang` alternates + localized structured data.

## RTL / LTR

- Direction derived from `LOCALES[code].dir` — never hardcoded.
- Layouts use **CSS logical properties** (`ps-*`, `pe-*`, `ms-*`, `me-*`,
  `start-*`, `end-*`) so the entire UI mirrors automatically for Arabic,
  including nav, mega menus and product cards.
- Fonts switch by script: `html[lang="ar"]` → IBM Plex Sans Arabic,
  `html[lang="en"]` → Inter (configurable via `theme.typography`).

## Strings

- **No hardcoded user-facing text in components.** All UI copy comes from
  `messages/<code>.json` via `useTranslations`/`getTranslations`.
- ICU plurals/selects supported (see `listing.itemsCount`, incl. Arabic dual/few).

## Localized *content* (from the backend)

Domain content is localized at the data layer: `LocalizedText` = a
`{ [locale]: string }` map. Product names, descriptions, categories,
collections, CMS pages, nav labels and homepage section titles all use it. The
`t(text, locale)` helper resolves the active locale with graceful fallback.
The admin dashboard will expose a language tab per editable field and flag
untranslated values.

## Adding a language (e.g. French)

1. Add `"fr"` to `locales` in `src/i18n/routing.ts`.
2. Add a `fr` entry to `LOCALES` in `src/config/locales.ts` (dir, label, font).
3. Add `messages/fr.json`.
4. Provide `fr` values in localized DB content (dashboard).

No component or architectural changes required.
